# Contributing

Thanks for your interest in improving the HubSpot Agent Tools Starter.

## Before you open a pull request

- Keep changes public-safe and generic.
- Do not commit secrets, tokens, refresh tokens, customer data, or internal URLs.
- Prefer placeholder values over environment-specific examples.
- Keep the project understandable for non-developers and AI coding tools.

## Good first contribution patterns

- improve setup instructions
- clarify README examples
- fix placeholder bugs
- improve metadata examples
- add small, well-documented starter patterns

## Pull request guidance

- explain the problem clearly
- describe the smallest useful change
- mention any security or deployment impact
- update docs if behavior changes
- avoid bundling unrelated edits

## Security reports

Do not open public issues for security vulnerabilities.
Use SECURITY.md for reporting guidance.
